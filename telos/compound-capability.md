# Compound capability
> The work should make future work easier

The win is not any single session. It is cognitive and technical, in the form of expertise encoded in foundations, built and strengthened over time.

Each library we finish becomes a tool for the next project. Each narrow utility removes a little repeated pain. Each clear module gives us a place to stop, ship, and build from. Cumulative, compounding.

The same is true of the worker. Even if a piece of software eventually dies, the compound effect of learning is ours to keep. We take the skill to the bank: the sharper taste, the better pattern, the clearer boundary, the improved instinct for what to ignore.

This is why [augmented theory-building][augmented-theory-building] matters more than output alone. A tool that leaves us dependent has given us a result. A tool that leaves us more capable has improved the system that will produce future results.

Small, boring tools are often the safest way to compound. Good [composable boundaries][composable-boundaries] are how a project turns into a platform for the next project.
