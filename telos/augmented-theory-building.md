# Augmented theory-building
> Use the machine to improve the theory

Software is leverage. Code is notation for thought.

The aim is not merely to produce more code, faster. The aim is to understand the thing better: the problem, the constraints, the structure of the solution, the boundaries between parts, the trade-offs worth accepting. The machine is useful when it helps us build that theory.

This is where [human agency][human-agency] matters. The machine can generate, compare, summarize, search, transform, and test, but it does not know what should matter to us. Direction, taste, responsibility, and final judgment remain human work.

Good tools compress the path from intent to tested reality. They shorten the loop between “I think this is what I mean” and “here is what actually happens.” That loop is where theory improves: assumption, experiment, observation, correction.

So we try to use AI not only to make more things, but to make better contact with the thing itself. That is the fundamental promise of [literate systems][literate-systems].

Ultimately, if the result is only more output, we probably missed the point, lost the plot. If the result is clearer thought, better boundaries, and stronger future judgment, we are closer to [compound capability][compound-capability].
