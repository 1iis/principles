# The new personal computer
> Context and capabilities around human intent

The future of personal computing is a context-shaped working environment organized around human intent.

It is not merely a device, an app launcher, or a chatbot. It is the place where files, tools, accounts, models, memories, agents, and capabilities become available in the right form for the work at hand.

## Personal means agency

A personal computer worthy of the name should augment the person.

That sounds obvious, but AI makes it easy to drift in the opposite direction: tools that act for us while making us less able to [understand][understanding-is-the-work], inspect, direct, or repair the work. That is not personal computing but delegated opacity: the opposite of [intelligible systems][intelligible-systems].

The point is not that the machine should do everything. The point is that the person should become more capable: better able to express intent, inspect consequences, combine tools, understand systems, and recover from mistakes.

The “personal” in personal computer is not ownership. It is [agency][human-agency].

## The user is now a pair

The new personal computer has to serve a strange pair: a human and an LLM.

The human is continuous in time, embodied, causal, responsible, and limited in attention. The model is vast in context, fluent in symbols, useful in bursts, but not continuous in the same way. It does not carry a life, a body, a history of consequences, or the same sense of risk.

Handled poorly, that mismatch becomes friction: lost context, misplaced confidence, opaque automation, and workflows that make the person weaker. Handled well, it becomes complementarity. The human supplies intent, judgment, continuity, and stakes. The machine supplies recall, transformation, synthesis, and execution across capabilities.

The new personal computer is the layer that organizes this pair. It sits above the operating system: not replacing files, applications, devices, or accounts, but bridging them semantically around the work the human is trying to do.

In that sense, it is a kind of hyper-system: a semantic layer above the operating system.

## A surface for theory-building

A personal computer is not only where work is executed. It is where the user [builds a theory of the work][theory-building].

The system should help answer: what is happening, why does it matter, what changed, what constraints apply, what can be tried next, and what did reality say when we tried it?

AI makes this more important, not less. If [generated work][generated-buffet] appears faster than understanding can form, the user loses the plot. The new personal computer should compress the path from intent to tested reality while preserving the reasoning that makes the result maintainable: [slow, then fast][slow-then-fast].

## Intent belongs in the medium

The new personal computer is [literate][literate-systems] by default.

Prose, code, data, outputs, tools, prompts, examples, decisions, and feedback should not live in disconnected silos. They should share a readable computational medium, or at least point to each other precisely enough that the user and the machine can recover the working context.

This is not about putting everything in one document. It is about making intent operational. The system should know where the explanation lives, where the [artifact][exploration-and-artifact] lives, what command produced the output, what decision survived, and what should not be reopened.

## Caps are the new apps

The old personal computer was organized around applications. Applications still matter, but they are no longer the whole story.

In AI-assisted work, the more useful unit is often a capability: inspect this file, run this check, summarize this history, translate this intent into a command, query this database, compare these options, deploy this artifact, retrieve this memory.

A capability may be packaged as an application feature, command, API, tool, or skill. What matters is that capabilities can [compose][composable-boundaries] into ad hoc workflows.

Operating systems and applications package capabilities. Agents can invoke capabilities. Humans compose capabilities. The personal computer becomes the environment where those capabilities are made available, bounded, inspectable, and useful.

This is the shift from apps to caps: not because apps disappear, but because intent increasingly cuts across them, and so the application unbundles under the hood, adopting an API shape.

## Context is the substrate

As tools, files, devices, accounts, models, memories, agents, and capabilities multiply, context becomes the substrate of personal computing.

The operating problem is no longer only where the file is, what opens it, or what command runs. It is how the right context and capabilities become available at the right moment without turning into clutter or opacity.

This system-level requirement warrants [context discipline][context-discipline].

## Capability should compound

A personal computer should get better with use.

Not by becoming mysterious or over-personalized in ways the user cannot inspect, but by accumulating useful, [boring][boring-is-good] structure: tools, examples, commands, notes, templates, histories, summaries, memories, conventions, and reusable capabilities.

Each solved problem should leave behind something more than an answer. It should leave a sharper tool, a clearer example, a better interface, a named workflow, or a reusable piece of context.

The environment should help [capability compound][compound-capability] without turning into clutter.

## The destination

The destination is not a smarter app launcher.

It is a humane, inspectable, composable working environment where context and capabilities meet human purpose.

It is personal computing as augmentation again: not the machine replacing the person, but the person becoming more capable through a medium that remembers, explains, composes, runs, and learns with them.
