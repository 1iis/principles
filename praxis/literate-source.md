# Literate source
> Keep why near how

Source is not just code. It is durable context.

The machine needs instructions, but the human needs intent: why this exists, what structure it has, what constraints matter, and how to return to it later without reconstructing everything from scratch. In AI-assisted work, the model benefits from the same proximity of meaning.

Explanations say why; instructions say how. A good artifact keeps them close enough to inform each other. The reason for a function should be near the function. The example that explains an interface should be near the interface. The output that validates a step should be near the step.

In the ideal case, one artifact can become source, documentation, tutorial, testbed, and memory at once. It can be read top-to-bottom for full comprehension, or targeted by section, symbol, example, or decision when only one part matters.

Structure is retrieval machinery. Headings, names, links, examples, signatures, and outputs let humans and machines land on the right context without first wandering through the whole project.

This is the source-level application of [context discipline][context-discipline]: intent belongs in the medium, but it has to be placed deliberately.

A clean source artifact can exist because [exploration happened elsewhere][exploration-and-artifact]. Written once, it may be read many times: by us, by collaborators, by future-us, and by the machines we ask to help.
